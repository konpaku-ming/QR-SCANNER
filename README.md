## 安装

### 系统要求

- **浏览器**：Microsoft Edge（Chromium，88+）或 Google Chrome（88+）
- **平台**：Windows / macOS / Linux
- **权限**：`activeTab`、`clipboardRead`（用于剪贴板图片识别）

### GitHub Releases 下载安装（推荐）

1. 前往 [GitHub Releases](https://github.com/OWNER/REPO/releases) 页面
2. 下载最新版本的 `qr-scanner-vX.X.X.zip`
3. 解压到任意文件夹（如 `~/qr-scanner/`）
4. Edge 浏览器打开 `edge://extensions/`，开启**开发者模式**
5. 点击**加载解压缩的扩展**，选择解压后的文件夹

> **注意**：由于本扩展使用 zxing-wasm（WebAssembly）作为解码引擎，需确保浏览器版本为 **Edge 88+** 或 **Chrome 88+**。

### 开发者模式加载（本地源码）

1. 克隆本仓库到本地
2. Edge 浏览器打开 `edge://extensions/`，开启**开发者模式**
3. 点击**加载解压缩的扩展**按钮
4. 选择仓库根目录（包含 `manifest.json` 的文件夹）
5. 扩展图标将出现在浏览器工具栏中

### GitHub Releases 下载安装

1. 前往 [GitHub Releases](https://github.com/OWNER/REPO/releases) 页面

> **注意**：由于本扩展使用 zxing-wasm（WebAssembly）作为解码引擎，需确保浏览器版本为 **Edge 88+** 或 **Chrome 88+**。

## 使用方式

### 1. 通过弹出窗口扫描（推荐）

1. 点击浏览器工具栏上的 **QR SCANNER** 图标，打开弹出窗口。
2. 点击**扫描当前页面**按钮。
3. 扩展截取当前视口，对整幅截图进行多二维码检测，识别视口内任意位置的二维码（包括 `<img>`、CSS 背景图、Canvas 等）。
4. 识别成功后，在二维码位置生成蓝色脉冲高亮框，扩展图标 Badge 显示当前页识别到的二维码数量。
5. 点击高亮框弹出操作菜单，可选择**打开链接**或**复制内容**。
6. 如需清除所有高亮标记，点击**清除标记**按钮。

### 2. 识别剪贴板（适合任意屏幕截图）

1. 使用系统截图工具截取包含二维码的屏幕区域：
   - Windows：按 **`Win + Shift + S`** 框选截图
   - macOS：按 **`Cmd + Shift + 4`** 框选截图
2. 截图会自动保存到剪贴板。
3. 点击工具栏上的 **QR SCANNER** 图标，打开 Popup。
4. 点击**识别剪贴板**按钮。
5. 扩展读取剪贴板中的图片，自动解码其中含有的**全部二维码**（支持同图多码）。
6. 识别成功后所有结果会保存到历史记录，可直接打开或复制。

> **适用场景**：PDF 文件、桌面应用、其他浏览器窗口、聊天记录中的二维码——任何能截图到剪贴板的内容。

### 3. 通过快捷键扫描

- 按下默认快捷键 **`Alt + Shift + Q`**（可在 `edge://extensions/shortcuts` 中修改）。
- 插件会立即对当前页面视口进行截图扫描，并渲染高亮标记。

### 4. 通过右键菜单扫描

1. 在网页上**右键点击**任意图片。
2. 选择上下文菜单中的**"扫描此图片中的二维码"**。
3. 插件会尝试对该图片进行解码：
   - 优先直接读取页面内的图片像素进行识别；
   - 若受跨域限制，则通过扩展后台 `fetch` 获取图片数据后再次解码。
4. 识别成功后会高亮该图片并显示 Toast 提示。

## 功能说明

### 已支持

| 功能 | 说明 |
|------|------|
| **整图多二维码检测** | 对当前视口整幅截图进行扫描，不依赖 `<img>` 标签，可检测页面内任意位置的二维码（包括 CSS 背景图、Canvas、SVG 等）。采用 zxing-wasm 原生多码检测，同图多码识别率高。 |
| **视口截图扫描** | 截取当前可见区域，对跨域图片也能识别二维码，不受 CORS 限制。 |
| **剪贴板多码识别** | 读取剪贴板图片，通过 zxing-wasm 原生多码检测识别图片中含有的全部二维码。支持 Win+Shift+S 等任意截图工具。 |
| **高亮 Overlay** | 识别成功后，在二维码位置显示半透明蓝色脉冲边框，层级最高（`z-index: 2147483647`），不会被页面元素遮挡。 |
| **操作菜单** | 点击 Overlay 弹出操作菜单，可选择**打开链接**或**复制内容**到剪贴板。 |
| **扫描历史记录** | 自动保存最近 50 条识别结果（二维码内容、页面标题、识别时间），关闭浏览器后仍保留。 |
| **Popup 最近识别** | 弹出窗口底部展示最近 5 条记录，支持快速打开、复制或删除单条。 |
| **右键单图扫描** | 右键菜单直接扫描单张图片；若遇跨域限制，自动通过后台 `fetch` 获取后解码。 |
| **扫描结果反馈** | 扫描完成后，扩展图标显示 Badge 计数（当前页识别到的二维码数量），同时页面顶部弹出 Toast 提示。 |
| **SPA 动态监听** | 通过 `MutationObserver` 监听 DOM 变化，单页应用路由切换或异步加载内容后自动重新扫描。 |
| **清除标记** | 一键移除页面上所有已渲染的高亮框和操作菜单。 |
| **快捷键支持** | 支持自定义快捷键触发扫描（默认 `Alt + Shift + Q`）。 |
| **底层解码引擎** | 采用 zxing-cpp WebAssembly 工业级解码器，单码/多码识别率、速度均优于纯 JS 方案。失败时自动 fallback 到 jsQR。 |

### 暂不支持

| 功能 | 说明 |
|------|------|
| **视口外二维码** | `captureVisibleTab` 只能截取当前可见区域，超出屏幕范围的二维码无法识别。请先滚动到目标区域再扫描。 |
| **浏览器内置页面扫描** | `chrome://`、`edge://`、`about:` 等浏览器内部页面（包括内置 PDF 查看器）无法注入 Content Script，因此无法渲染 Overlay。 |
| **自动扫描** | 不会随页面加载自动识别，每次需要手动触发。 |
| **设置页面** | 选项页面（设置）仅为占位 UI，暂无可调节的配置项。 |

## 已知限制

1. **识别范围仅限视口**  
   扫描基于当前屏幕可见区域截图。如果二维码位于页面下方或右侧（需要滚动才能看到），请先滚动至该区域，再执行扫描。

2. **浏览器内置页面无法扫描**  
   `chrome://`、`edge://`、`about:` 等浏览器内部页面（包括 Edge 内置 PDF 查看器）不允许注入 Content Script，因此扩展无法在这些页面上渲染高亮 Overlay。如需识别 PDF 中的二维码，请使用「识别剪贴板」功能截图后识别。

3. **二维码质量影响识别率**  
   过于模糊、压缩失真、尺寸过小或被严重遮挡的二维码可能识别失败。虽然已集成自适应阈值预处理，但极端低质量图像仍可能无法检出。

## 隐私说明

- **扫描触发方式**：用户可通过 Popup 按钮、快捷键、右键菜单或框选区域手动触发扫描。若页面为 SPA（单页应用），扩展还会通过 `MutationObserver` 在 DOM 内容变化后自动重新扫描视口内的二维码。自动扫描仅在当前页面可见时触发，且受 3 秒最小间隔限制。
- **截图仅用于本地解码**：`captureVisibleTab` 截取的视口图像仅在本地内存中处理，不会上传至任何服务器。
- **WASM 离线运行**：zxing-wasm 的 `.wasm` 文件打包在扩展内部，通过 `chrome-extension://` 协议本地加载，无需连接外部 CDN。
- **无数据收集**：扩展不使用任何分析、统计或远程 API，所有操作均在本地完成。

## 测试

### Node.js 单元测试（纯逻辑）

测试内容涵盖：字符串截断、截图坐标映射、图片过滤逻辑、jsQR 加载与基本行为，以及 `qr-decoder.js` 纯函数（`extractRegion`、`eraseQRRegion`、`fastAdaptiveThreshold`、`truncate`）。`qr-decoder.js` 作为 zxing-wasm 的 fallback 被保留，这些测试确保 fallback 路径仍然可用。

```bash
node tests/node-unit-test.js
```

### 浏览器单元测试（需网络）

在仓库根目录启动本地 HTTP 服务器：

```bash
python3 -m http.server 8765
```

然后浏览器访问：

- **纯逻辑测试**：http://localhost:8765/tests/unit-test.html
  - 自动测试 truncate、坐标映射、过滤逻辑、jsQR 解码、Toast DOM 操作。
- **手动功能测试**：http://localhost:8765/tests/manual-test.html
  - 包含多个二维码场景（同域/跨域、小图过滤、右键菜单、视口外二维码等），需在 Edge 中加载扩展后人工验证。
- **SPA 端到端测试**：http://localhost:8765/tests/e2e-spa.html
  - 模拟 SPA 路由切换和动态内容加载，测试 MutationObserver 自动重扫功能。包含路由切换、批量插入二维码、异步加载等场景。
